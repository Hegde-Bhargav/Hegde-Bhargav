## 🚀 Verification Projects

<details>
<summary><b>🔷 RISC-V RV32I Processor Core Verification (UVM)</b> — click to expand</summary>
<br>

> Full UVM testbench built from scratch for open-source RV32I core verification.

- 🏗️ Developed UVM environment: **agent, driver, monitor, scoreboard** for RV32I pipeline
- 🔍 Debugged pipeline execution across **fetch → decode → execute → memory → writeback** stages
- 🛡️ Implemented **SVA checkers** for illegal instructions, memory violations, PC continuity
- 🐛 Identified false-pass scenarios from data hazards; resolved via SVA stall checks + scoreboard cross-verification
- 📊 Achieved **~95% functional coverage**

`UVM` `SystemVerilog` `RISC-V` `SVA` `QuestaSim` `Waveform Analysis`

</details>

<details>
<summary><b>🔶 UART Protocol Verification (SystemVerilog)</b> — click to expand</summary>
<br>

> Self-checking SystemVerilog testbench for UART transceiver across multiple configurations.

- 🎲 Randomized **baud rates, parity modes** (even/odd/none), and **data widths** (5–9 bits)
- ✅ Scoreboard auto-comparing transmitted vs received data
- 🐞 Traced & fixed subtle baud-rate mismatch causing intermittent failures at **115200 baud**
- 💥 Edge cases: frame errors, parity errors, buffer overrun, back-to-back transactions

`SystemVerilog` `Constrained Random` `Scoreboard` `ModelSim` `Coverage`

</details>

<details>
<summary><b>🔵 SPI Master-Slave Interface Verification</b> — click to expand</summary>
<br>

> Verification environment with full CPOL/CPHA mode coverage for SPI communication.

- 🔄 Tested all **4 SPI modes**, word lengths (8/16/32-bit), LSB/MSB-first configurations
- 🛡️ SVA checkers for **chip-select timing** and setup/hold violations
- 🧪 Randomized stress tests for back-to-back transfers; **validated on FPGA with logic analyzer**

`SPI` `SVA` `FPGA` `SystemVerilog` `Randomization`

</details>

<details>
<summary><b>🟣 FIFO Buffer Verification with Coverage Analysis</b> — click to expand</summary>
<br>

> Synchronous and asynchronous FIFO verification with CDC behavior focus.

- 🔁 Tested full/empty/almost-full/almost-empty flags, pointer wrap-around, simultaneous R/W
- ⚠️ Identified **metastability window** during back-to-back writes across clock domains
- 🛡️ Added **synchronizer checks** for CDC handshaking validation

`FIFO` `CDC` `Async Design` `Coverage` `Stress Testing`

</details>

<details>
<summary><b>⚙️ ALU & Register File Verification</b> — click to expand</summary>
<br>

> Directed + constrained random testbench for 32-bit ALU.

- ✅ Covered all **arithmetic, logical, and shift operations** with register file read/write hazard testing
- 🛡️ Assertion checks for overflow, underflow, and zero-flag generation
- 🐛 Debugged edge case: simultaneous R/W to same register returning **stale data**

`ALU` `32-bit` `ABV` `Directed Testing` `ModelSim`
